2.2. Elementgruppen

Um die Elemente leichter zu beschreiben, sind sie in spezifischen Gruppen organisiert.

2.2.1. Struktur-Elemente-Gruppe

Struktur-Elemente werden für die Beschreibung der Struktur genutzt und bilden den strukturellen Rahmen der Datenstruktur.

Vertreter:

  1. celem

  2. choice

  3. cid

  4. elem

  5. fixseq

  6. ref

  7. seq

2.2.2. Daten-Elemente-Gruppe

Daten-Elemente beinhalten keine anderen Elemente und sind Container für die Daten.

Vertreter:

  1. bin

  2. bin64

  3. binHex

  4. formatnum

  5. formatstr

  6. num

  7. str

2.2.3. Zeit-Elemente-Gruppe

Zeit-Elemente beinhalten keine anderen Elemente und sind Container für Zeitangaben und/oder Datumsangaben.

Vertreter:

  1. date

  2. datetime

  3. formattime

  4. time

2.2.4. Ausdrucks-Elemente-Gruppe

Ausdrucks-Elemente definieren finale Ausdrücke oder zu bewertende Konstrukte.

Vertreter:

  1. const

  2. sxp

3. Elemente

3.1. bin

Ein Element, welches binäre Daten beinhaltet.

3.1.1. Eigenschaften

  1. Es wird eine byteOrder angegeben.

  2. Eine Kodierung der Daten kann über encoding angegeben werden (z.B. Base32, Base64, Base85).

  3. Mime Media Typ kann über das mime Attribut angegeben werden.

Definition
<bin byteOrder="littleEndian" id="ID1"/>
<bin encoding="Base64" id="ID2"/>
<bin mime="text/plain" id="ID3"/>
Erlaubte Attribute

db-column-name, Generische Attribute, byteOrder, mime

3.2. bin64

Dieses Element enhält Binärdaten, die via Base64 kodiert sind.

Erlaubte Attribute

db-column-name, Generische Attribute

3.3. binHex

Ein Elemente, welches hexadezimal kodierte Daten enthält.

Erlaubte Attribute

db-column-name, Generische Attribute

3.4. celem

Ein Choice-Container-Element definiert die kleinste zu beschreibende Einheit innerhalb einer Choice. Es ist rekursiv definiert, d.h. es kann andere Elemente enthalten.

<choice id="card">
  <celem id="row" s="semantic">
    <num id="row_num"/>
    <str id="row_str"/>
  </celem>
</choice>

  1. Ein einfaches Choice-Container-Element enthält keinen Wert!

  2. Ein einfaches Choice-Container-Element kann andere Elemente enthalten.

  3. Ein Choice-Container-Element kann nur direkt unterhalb von choice Elementen vorkommen.

Erlaubte Attribute

Generische Attribute

Erlaubte Elemente

celem, cid, Datenelemente, Ausdruckselement

3.5. choice

Ein Element, mit dem Wahlmöglichkeiten in der Struktur abgebildet werden können, so dass die Daten nicht von vornherein ein bestimmtes Format haben müssen.

  1. Matching einer Struktur oder von Elementen

  2. Innerhalb einer Choice müssen Elemente in einem oder mehreren celem stehen.

  3. Die Reihenfolge der Datenelemente bestimmt das Matching. Daher sollten spezifische Datenelemente vor einem str Element definiert werden.

  4. Das letzte Daten-Element einer Choice, die sich innerhalb einer Sequenz befindet, darf kein stop-sign enthalten.

Definition
<choice id="card">
  <celem id="row1">
    <num id="row1_num" start-sign="\d" stop-sign=";"/>
    <str id="row1_str" start-sign="NAME" stop-sign=":"/>
  </celem>
  <celem id="row2">
    <num id="row2_num" start-sign="\d" stop-sign=";" />
    <str id="row2_str" start-sign="NAME"/>
  </celem>
</choice>
Erlaubte Attribute

Generische Attribute

Erlaubte Elemente

celem

3.6. cid

Ein Element, welches als Schachtelelement für ein Daten-Element genutzt werden kann.

  1. Eine benutzerdefinierte ID dient als Schachtelelement für ein Datenelement des Typs string oder numerical.

  2. Eine benutzerdefinierte ID kann eine Klasse (class) definieren.

Definition
<elem id="someElement">
  <cid id="myCustomID" class="myCustomClass">
    <str/>
  </cid>
  <str id="ID"/>
</elem>

<seq id="someList" min="2">
  <elem id="structure">
    <cid id="anotherCustomID" class="nestedClass">
      <str id="ID"/>
    </cid>
    <str id="anotherID"/>
  </elem>
</seq>
Erlaubte Attribute

Generische Attribute

Erlaubte Elemente

Datenelemente

3.7. const

Eine Konstante ist ein Schachtelelement für genau ein Element aus dem Bereich Datenelemente.

Definition
<const id="foo">
  <str id="fooStr">Foo</str>
</const>

<const id="bar">
  <num id="barNum">123</num>
</const>
Erlaubte Attribute

Generische Attribute

Erlaubte Elemente

Datenelemente

3.8. date

Ein Element, welches ein Datum beschreibt. Das Datum muß im ISO-Format (yyyy-MM-dd) vorliegen!

Definition
<date id="dateField/>
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, xml-attribute-name, xml-attribute-parent, xml-element-name

3.9. datetime

Ein Element, das ein komplettes Datum inklusive Zeit (Zeitstempel) beschreibt. Der Zeitstempel muß im ISO-Format vorliegen!

Definition
<datetime id="dateTime"/>
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, xml-attribute-name, xml-attribute-parent, xml-element-name

3.10. dfasdl

Das Wurzelelement eines DFASDL-Dokuments beinhaltet Attribute, die das Dokument beschreiben.

  1. Es darf nur einmal im gesamten Dokument und zwar auf der obersten Ebene vorkommen.

  2. Der genutzte semantische Raum wird im Attribut semantic definiert.

  3. Das Attribute default-encoding kann genutzt werden, um eine Vorgabe für nicht definierte encoding Attribute an einzelnen Elementen zu setzen.

Definition
<dfasdl xmlns="http://www.dfasdl.org/DFASDL" semantic="niem">
  ...
</dfasdl>
Erlaubte Attribute

Wurzelattribute

Erlaubte Elemente

Alle Elemente außer dem Wurzelelement (dfasdl).

3.11. elem

Ein Element definiert die kleinste Einheit innerhalb eines Datenformats. Es ist rekursiv definiert, d.h. es kann andere Elemente enthalten.

  1. Ein einfaches Element enthält keinen Wert!

  2. Ein einfaches Element kann andere Elemente enthalten.

Definition
<elem id="foo">
  <seq id="bar" max="2">
    <str id="foobar"/>
  </seq>
</elem>
<elem id="empty"/>
Erlaubte Attribute

Generische Attribute

Erlaubte Elemente

Alle Elemente außer dem Wurzelelement (dfasdl).

3.12. fixseq

Eine fixe Sequenz gibt an, daß ihre Kindstruktur sich wiederholt und eine endliche Menge an Elementen beinhaltet.

  1. Eine fixseq hat die gleichen Eigenschaften, wie eine seq, mit der Ausnahme, dass man eine genaue Angabe der Anzahl der beinhalteten Elemente machen muß.

  2. Zur Angabe der Anzahl der vorhandenen Elemente wird das count Attribut verwendet.

  3. Durch das stop-sign kann eine Zeichenkette angegeben werden, welche die Sequenz beendet. Wenn dieses stop-sign in den Daten auftaucht, wird die Sequenz beendet und die Abarbeitung weiterer Elemente, welche nach der Sequenz kommen, fortgesetzt.

Definition
<fixseq id="accountList" count="2">
  <elem id="account">
    <str id="number"/>
  </elem>
</fixseq>
Erlaubte Attribute

Generische Attribute, db-select, db-insert, db-primary-key, db-update, count, keepID

Erlaubte Elemente

Strukturelemente, Datenelemente

3.13. formatnum

Ein numerisches Datenelement, das dem angegebenen Format (format) genügen muß.

Die folgenden Zeichen sind zulässig sowohl in den Daten als auch in der Wertangabe des Attributs defaultnum:

  1. Minus (-)

  2. Ziffern (0-9)

  3. Punkt (.)

  4. Komma (,)

Definition
<formatnum format="(\d\d\d)" id="ID" max-digits="12"/>
<formatnum decimal-separator="." format="([0-9]{1,3}\.\d{1,2})" id="ID2"
max-digits="3" max-precision="2"/>
Der Standardwert von decimal-separator ist das Komma (,). Ist kein Wert für decimal-separator angegeben, wird dieser Standardwert genommen.
Der zu matchende Teil des format Attributes muss innerhalb einer Gruppe (…​) stehen!
Erlaubte Attribute

Generische Attribute, db-auto-inc, db-column-name, db-foreign-key, decimal-separator, defaultnum, format, max-digits, max-precision, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

Wenn das Dezimaltrennzeichen behalten werden soll, muss es über das Attribut decimal-separator angegeben werden!

3.14. formatstr

Ein Element für Zeichenketten, welches das Format (format) definieren muß.

Definition
<formatstr id="formatA" format="(\w\w\d)"/>
<formatstr id="formatB" format="(\w{1,10})"/>
<formatstr id="formatC" format=".*?:(.*)"/>
Der zu matchende Teil muss innerhalb einer Gruppe (…​) stehen!
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, defaultstr, format, trim, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

3.15. formattime

Für Datums- und Zeitangaben, die nicht der ISO-Notation entsprechen, kann dieses Element verwendet werden. Die Angabe für das Attributformat muß einen für den Java DateTimeFormatter verarbeitbaren Wert enthalten!

Definition
<formattime id="my-time-is-now" format="dd.MM.yyyy HH:mm:ss X"/>
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, defaultstr, format, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

3.16. num

Ein Datenelement, das einen numerischen Wert beinhaltet.

  1. Ein numerisches Element darf nur Ziffern und gegebenenfalls ein Minuszeichen als Startzeichen enthalten.

  2. Ein numerisches Element kann eine exakt einzuhaltende Ziffernanzahl (length) definieren.

    1. Das Minuszeichen wird bei der Berechnung der Länge nicht berücksichtigt.

  3. Ein numerisches Element kann eine maximal zu berücksichtigende Ziffernanzahl (max-digits) definieren.

    1. Das Minuszeichen wird bei der Anzahl der maximalen Ziffernanzahl nicht berücksichtigt.

  4. Ein numerisches Element kann die Anzahl der Nachkommastellen (precision) definieren.

  5. Ein numerisches Element kann einen Standardwert (defaultnum) definieren, der bei fehlenden Datensätzen eingefügt wird.

Die folgenden Zeichen sind zulässig sowohl in den Daten als auch in der Wertangabe des Attributs defaultnum:

  1. Minus (-)

  2. Ziffern (0-9)

Definition
<num id="numberA" length="4"/>
<num id="numberB" max-digits="5"/>
<num id="Pi" length="10" precision="9" defaultnum="3141592653"/>
Erlaubte Attribute

Generische Attribute, db-auto-inc, db-column-name, db-foreign-key, defaultnum, length, max-digits, precision, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

3.17. ref

Eine Referenz bezieht sich auf ein im Dokument definiertes Datenelement, welches an Stelle der Referenz eingesetzt wird.

  1. Eine Referenz muß eine Quellen-ID (sid) definieren, welches der id des referenzierten Datenelementes entspricht!

  2. Das referenzierte Datenelement muss vor der Referenz im DFASDL Datenbaum vorkommen.

  3. Wenn eine Referenz in einer Sequenz definiert wird, muss sie am Ende der Sequenz stehen.

  4. In einer Sequenz darf nur eine Referenz vorkommen.

  5. Wenn für eine Referenz keine semantische Bedeutung (s) definiert ist, wird die Semantik des referenzierten Eintrags übernommen!

Definition
<elem id="someBlockElement">
  <elem id="anotherID">
    <str id="firstname"/>
    <str id="lastname"/>
    <num id="mainNumber"/>
  </elem>
</elem>
<ref id="number" sid="mainNumber"/>
<!-- Referenzieren aus einer Sequenz -->
<seq id="accountList" max="999">
  <elem id="account">
    <num id="account_id"/>
    <str id="name"/>
    <str id="account"/>
    <seq id="children">
      <elem id="alter">
        <num id="anzahl"/>
        <num id="age"/>
        <ref sid="account_id" id="children_account_id">
      </elem>
    </seq>
  </elem>
</seq>
Erlaubte Attribute

Generische Attribute, sid, value

3.18. seq

Ein Sequenzelement definiert sich wiederholende Strukturen.

  1. Eine Sequenz kann die folgenden Varianten definieren:

    1. eine Mindestanzahl (min)

    2. eine Maximalanzahl (max)

    3. eine Mindest- und Maximalanzahl (min und max)

    4. keine Angabe (entspricht einer unendlichen Sequenz)

  2. Innerhalb einer Sequenz werden bei der Konvertierung keine IDs mit kopiert, sondern in das Attribut class kopiert. Aus der ID foo wird hierbei die Ausprägung id:foo im Klassenattribut.

  3. Wenn die IDs gelöscht werden sollen, muß das Attribut keepID mit false angegeben werden.

  4. Daten-Elemente müssen innerhalb einer Sequenz mit einem elem Element umfasst werden.

  5. Durch das stop-sign kann eine Zeichenkette angegeben werden, welche die Sequenz beendet. Wenn dieses stop-sign in den Daten auftaucht, wird die Sequenz beendet und die Abarbeitung weiterer Elemente, welche nach der Sequenz kommen, fortgesetzt.

  6. Das Attribut filter erlaubt die Definition eines Filters auf den Quelldaten. Lediglich Daten, die diesen Filter erfüllen werden berücksichtigt.

Definition
<seq id="accountList" min="42" max="999">
  <elem id="account">
    <str id="number" class="foo"/>
    <str id="name"/>
  </elem>
</seq>

<seq id="accountList2" keepID="false">
  <elem id="account">
    <str id="number" class="bar"/>
  </elem>
</seq>

<seq id="salaries" filter="salary > 20000">
  <elem id="employee">
    <str id="name"/>
    <num id="salary"/>
  </elem>
</seq>
Erlaubte Attribute

Generische Attribute, db-select, db-insert, db-primary-key, db-update, min, max, keepID, filter

Erlaubte Elemente

Strukturelemente, Datenelemente

3.19. str

Ein Datenelement für Zeichenketten. Dieses Element kann, aber sollte nicht, als generischer Container genutzt werden, da sich fast alle Daten in Form von Zeichenketten darstellen lassen.

  1. Ein Zeichenelement darf nur Zeichen in der Standard- oder eingestellten Kodierung enthalten.

  2. Ein Zeichenelement kann die Kodierung der zu erwartenden Zeichenfolge (encoding) definieren.

  3. Ein Zeichenelement kann eine exakt einzuhaltende Anzahl (length) von Zeichen definieren.

  4. Ein Zeichenelement kann eine maximal zu berücksichtigende Zeichenanzahl (max-length) definieren.

  5. Ein Zeichenelement kann ein Folge von Zeichen definieren, die als Stopzeichen (stop-sign) gewertet werden.

  6. Ein Zeichenelement kann einen Standardwert (defaultstr) definieren, der bei fehlenden Datensätzen eingefügt wird.

Definition
<str id="A" encoding="UTF-16"/>
<str id="B" length="3"/>
<str id="C" max-length="5"/>
<str id="possiblyEmpty" defaultstr="missingValue"/>
<str id="D" stop-sign="\n"/>
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, defaultstr, length, max-length, trim, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

3.20. sxp

Ein Element, das eine Scala-Expression enthält.

Dieses Element wird entfernt!

  1. Der Ausdruck entspricht einer Scala-Expression

  2. Der Ausdruck muß beginnen mit: <![CDATA[{

  3. Der Ausdruck muß enden mit: }]]>

  4. Innerhalb des sxp Elementes dürfen alle gängigen XML Elemente eingefügt werden.

Definition
<sxp id="expOne">
  <ul><![CDATA[{List(apple, banana, orange).map(i => <li>{i}</li>)}]]></ul>
</sxp>
Erlaubte Attribute

Generische Attribute

3.21. time

Ein Datenelement für Zeitdaten, die der ISO-Notation entsprechen müssen.

Definition
<time id="high-noon"/>
Erlaubte Attribute

Generische Attribute, db-column-name, db-foreign-key, unique, xml-attribute-name, xml-attribute-parent, xml-element-name

4. Attribute

4.1. Wurzelattribute

Wurzelattribute sind nur am Wurzelelement dfasdl erlaubt.

4.1.1. default-encoding

Eine Vorgabe für die Kodierung der eingelesenen Daten. Es muß sich um eine valide Definition z.B. utf-8 handeln.

Dieses Attribut ist sinnvoll, wenn alle oder die meisten Elemente die gleiche Kodierung verwenden.

4.1.2. semantic

Der Inhalt des Attributs beschreibt den genutzten semantischen Raum des Dokuments. Derzeit sind die folgenden Werte erlaubt: custom, niem, udef.

4.2. Generische Attribute

Generische Attribute sind an allen Elementen außer dem Wurzelelement verfügbar.

4.2.1. class

Definiert eine Klassenbezeichnung für ein Element.

4.2.2. correct-offset

Durch dieses Attribut kann der Offset der eingelesenen Daten korrigiert werden.

Der Offset kann in die positive und auch in die negative Richtung korrigiert werden.

4.2.3. encoding

Die verwendete Kodierung der eingelesenen Daten. Es muß sich um eine valide Definition z.B. utf-8 handeln.

4.2.4. id

  1. Eine ID ist eine Zeichenkette.

  2. Eine ID muß mit einem Buchstaben beginnen.

  3. Eine ID darf Buchstaben aus dem ASCII-Alphabet, Zahlen und Unterstriche sowie Minuszeichen enthalten.

  4. Eine ID darf innerhalb eines Dokuments nur einmal vorkommen.

  5. Wenn nicht anders angegeben, ist die Angabe der ID für alle Elementtypen zwingend erforderlich!

Sollte keine ID angegeben sein, wird später automatische eine erzeugt. Es ist aber ratsam selber eine eigene ID zu vergeben, um das Mapping besser manuell nachkorrigieren zu können.

4.2.5. s

Beschreibt die semantische Bedeutung des Elements.

  1. Die semantische Bedeutung wird als Zeichenkette angegeben.

  2. Es sind nur Werte zulässig, die dem gewählten semantischen Standard entsprechen.

4.2.6. start-sign

Ein regulärer Ausdruck, der den Begin der Elementdaten beschreibt.

Ein start-sign darf nicht leer sein!

4.2.7. stop-sign

Ein regulärer Ausdruck, der das Ende der Elementdaten beschreibt.

Das Standard stop-sign berücksichtigt UNIX und Windows Zeilenenden und ist folgendermaßen definiert: \r\n?|\n
Ein stop-sign darf nicht leer sein!

4.3. Elementspezifische Attribute

Es gibt Attribute, die nur an bestimmten Elementen erlaubt sind.

4.3.1. byteOrder

Gibt die Sortierung der Bytes für Binärdaten an. Die folgenden Angaben sind möglich:

  1. bigEndian

  2. littleEndian

  3. middleEndian

Erlaubt an folgenden Elementen

bin

4.3.2. count

Definiert eine Anzahl. Erlaubt ist ein numerischer Wert größer oder gleich Null.

Erlaubt an folgenden Elementen

fixseq

4.3.3. db-auto-inc

Die Spalte des Elements in der Datenbank ist eine Autoinkrementspalte, d.h. deren Wert wird automatisch befüllt, wenn kein Wert für die Spalte angegeben wird.

Erlaubt an folgenden Elementen

formatnum, num

Da Datenbanken hinsichtlich der Verwendung von Autoinkrementspalten limitiert sind, sollte dieses Attribut nur an einem einfachen num Element ohne die Attribute precision und length verwendet werden!
Beispiel
<seq id="companies">
  <elem id="companies-row">
    <num id="companies-row-id" db-column-name="id" db-auto-inc="true"/>
  </elem>
</seq>

4.3.4. db-column-name

Der Spaltenname des Elementes in der Datenbank. Wenn dieser nicht angegeben ist, wird die ID des Elementes für den Spaltennamen herangezogen.

Erlaubt an folgenden Elementen

Dieses Attribut ist an allen Datenelementen gestattet.

4.3.5. db-foreign-key

Die Fremdschlüsseldefinition in der Datenbank für die Spalte, die durch das aktuelle Element beschrieben wird. Die Angabe erfolgt durch eine Zeichenkette, die kommasepariert die IDs aller DFASDL-Elemente enthält, die die jeweiligen Spalten beschreiben.

Erlaubt an folgenden Elementen

date, datetime, formatnum, formatstr, formattime, num, str, time

Beispiel
<seq id="companies">
  <elem id="companies-row">
    <num id="companies-row-id" db-column-name="id"/> (1)
    ...
  </elem>
</seq>

<seq id="contacts">
  <elem id="contacts-row">
    ...
    <num id="contacts-row-company-id" db-column-name="company_id" db-foreign-key="companies-row-id"/> (2)
  </elem>
</seq>

[1] Die Beschreibung einer Datenbankspalte. [2] Der Verweis via db-foreign-key auf das referenzierte Element.

4.3.6. db-insert

Möglichkeit zur Durchführung datenbankspezifischer INSERT-Statements, um Datensätze in eine Tabelle einzufügen. Die Syntax muß der für prepared Statements entsprechen.

Beispiel
INSERT INTO mytable (column1, column2) VALUES(?, ?)
Es ist möglich datenbankspezifische SQL-Syntax zu verwenden.
Erlaubt an folgenden Elementen

fixseq, seq

4.3.7. db-primary-key

Definiert den Primärschlüssel für eine Datenbanktabelle. Insofern das Attribut definiert ist, muß es mindestens einen oder mehrere durch Kommas getrennte Spaltennamen enthalten.

Die Spaltennamen müssen den Datenbankspaltennamen entsprechen!
Erlaubt an folgenden Elementen

fixseq, seq

4.3.8. db-select

Möglichkeit zur Durchführung datenbankspezifischer SELECT-Statements, um Datensätze aus einer Tabelle zu ermitteln.

Beispiel
SELECT
  x374 AS column1,
  y478 AS column2
FROM x2 JOIN y3 ON x2.id = y3.refId
WHERE x2.x23 = 1
ORDER BY y3.y1 ASC
Beispiel DFASDL
<?xml version="1.0" encoding="UTF-8" standalone="no"?>

4.3.9. db-update

Möglichkeit zur Durchführung datenbankspezifischer UPDATE-Statements, um Datensätze in einer Tabelle zu aktualisieren. Die Syntax muß der für prepared Statements entsprechen.

Beispiel
UPDATE mytable SET id = ?, column1 = ?, column2 = ? WHERE id = ?
Beispiel DFASDL
<?xml version="1.0" encoding="UTF-8"?>
<dfasdl xmlns="http://www.dfasdl.org/DFASDL" semantic="custom">
  <seq id="people" db-primary-key="id" db-update="UPDATE people SET id = ?, name = ?, time = now() WHERE id = ?">
    <elem id="people_row">
      <num db-column-name="id" id="id" max-digits="5"/>
      <str db-column-name="name" id="name" max-length="12"/>
    </elem>
  </seq>
</dfasdl>
Es ist möglich datenbankspezifische SQL-Syntax zu verwenden.
Erlaubt an folgenden Elementen

fixseq, seq

4.3.10. decimal-separator

Gibt das Dezimaltrennzeichen für ein numerisches Datenelement an. Erlaubt sind die folgenden Werte:

  1. Punkt (.)

  2. Komma (,)

  3. Momayyez (٫)

Erlaubt an folgenden Elementen

formatnum

4.3.11. defaultnum

Definiert einen Standardwert für ein numerisches Datenelement, für den Fall, daß die Daten leer sind. Das erlaubte Format ist abhängig vom konkreten numerischen Datenelement.

Erlaubt an folgenden Elementen

formatnum, num

4.3.12. defaultstr

Definiert eine Zeichenkette als Standardwert für ein Datenelement, für den Fall, daß die Daten leer sind.

Erlaubt an folgenden Elementen

formatstr, formattime, str

4.3.13. filter

Definiert einen Filterausdruck, der die zur Verfügung stehenden Daten eingrenzt.

Filter werden momentan nur auf Datenbanken unterstützt!
Sonderzeichen, die innerhalb von XML Probleme bereiten können wie z.B. < und & müssen korrekt maskiert werden!
Maskierte Sonderzeichen im Ausdruck
<seq id="foo" filter="my-column-data &lt; 1024">
  ...
</seq>
Erlaubt an folgenden Elementen

seq

4.3.14. format

Enthält eine Formatangabe für den Inhalt eines Datenelements.

Der zu matchende Teil muss innerhalb einer Gruppe () stehen!
Erlaubt an folgenden Elementen

formatnum, formattime, formatstr

4.3.15. length

4.3.16. keepID

Gibt an, ob die Werte des Attributs id innerhalb von Sequenzen erhalten werden sollen. Sie werden dann im Attribut class gespeichert. Erlaubt sind die Angaben true und false.

Per Standard ist dieses Attribut aktiviert, d.h. auf true gesetzt.

4.3.26. trim

Gibt an, ob für eingelesene Zeichenkette eine Säuberung durchgeführt werden soll. Hierbei werden Leerzeichen, Tabulatoren und Zeilenumbrüche entfernt. Die folgenden Angaben sind möglich:

left

Nur am Anfang der Zeichenkette säubern.
right

Nur am Ende der Zeichenkette säubern.
both

Am Anfang und am Ende der Zeichenkette säubern.
Erlaubt an folgenden Elementen

formatstr, str

4.3.27. unique

Definiert eine eindeutige Merkmalsausprägung für das Element im Datensatz. Wenn das Attribut vorhanden und auf true gesetzt ist, dann darf eine konkrete Merkmalsausprägung innerhalb eines Datensatzes nur einmal vorkommen. Dies ist analog zu einem UNIQUE CONSTRAINT in einer relationalen Datenbank.

Erlaubt an folgenden Elementen

formatnum,formatstr,formattime,num,str,time

4.3.28. value

Definiert eine Merkmalsausprägung eines Datensatzes.

Erlaubt an folgenden Elementen

ref

4.3.29. xml-attribute-name

Definiert den Namen des Attributs, welches am XML-Element (definiert über xml-attribute-parent) vorhanden ist. Dies dient dazu, Daten auslesen zu können, die in XML-Attributen stehen.

Definition
<seq id="foo">
  <elem id="row">
    <num id="age" xml-attribute-name="age" xml-attribute-parent="raw-data"/>
    <num id="count" xml-attribute-name="count" xml-attribute-parent="raw-data"/>
  </elem>
</seq>

4.3.30. xml-attribute-parent

Definiert den Namen eines XML-Elements, das Attribute beinhaltet, die ausgelesen werden sollen (siehe xml-attribute-name).

Definition
<seq id="foo">
  <elem id="row">
    <num id="age" xml-attribute-name="age" xml-attribute-parent="raw-data"/>
    <num id="count" xml-attribute-name="count" xml-attribute-parent="raw-data"/>
  </elem>
</seq>

4.3.31. xml-element-name

Falls der Name des auszulesenden XML-Elements von der id abweicht, kann er hiermit definiert werden (analog zu db-column-name).

Definition
<str id="some-id" xml-element-name="an-xml-id"/>
